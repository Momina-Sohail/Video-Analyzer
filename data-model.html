Compile metadata, transcription segments, summary breakdowns, and key moments into unified relational tables.

VideoJob Postgres Structure

Represents the master record for each video submission, mapping processing logs and fallback providers.

Column Type Indexes & Keys
id VARCHAR(255) PRIMARY KEY
user_id VARCHAR(255) INDEX (Foreign Key to User)
status VARCHAR(50) INDEX (queued | extracting | complete)
provider_used VARCHAR(50) groq | gemini (fallback log)
duration_seconds INTEGER Nullable (fetched via yt-dlp)

Prisma Schema

Shared relational definition for both Next.js app router and worker queue consumer.

model Transcript {
id         String   @id @default(uuid())
jobId      String   @unique
rawText    String
segments   Json
provider   String
job        VideoJob @relation(fields: [jobId])
}
